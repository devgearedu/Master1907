제목 : 환자 예약관리 프로그램
설명 : 간호사가 환자정보를 관리하고 입장을 시킨 뒤 의사의 처방에 따라
수납을 할 수 있다.
사전 조건 : 1. 환자 정보를 수정하면 모든 그리드에 적용이 되야 함.
2. 병원 업무의 절차에 맞게 프로그램이 잘 작동해야 함.
3. 현재 상태에 따라 아무것도 아닐 때 0, 예약중 10, 진찰중 20, 수납중 30 으로 구분
4. 예약 시간이 겹치거나 근무시간 외에 예약 불가해야 함.

테스트 절차 : 
v1. 서버 실행시 서버 폼 나타남.
2. 프로그램 실행시 스플래쉬 화면 나타난 뒤 메인 폼 나타남.
v3. 상단 패널에 예약 및 접수, 환자목록, 진료내역, 문자전송, 약품목록, 진찰 버튼 나옴.

v3. 예약 및 접수 버튼을 클릭하면 띄웠던 폼 내용이 사라지고 패널에 예약폼 띄우기
v4. 예약폼은 예약한 환자들의 목록을 띄움.
v5. 콤보박스에서 환자명, 예약일, 담당의사를 선택할 수 있음
v6. 에디트에 타이핑하면 콤보박스에 작성된 내용이 검색됨.
v7. 예약 추가버튼을 누르면 예약추가폼으로 넘어감.
v8. 예약 취소버튼을 누르면 YesNo 물은 뒤 Yes 누르면 선택된 환자의 예약일이 삭제됨
v9. 입장 버튼을 누르면 선택된 환자가 진찰중 상태(20)로 변경됨.(이후 의사폼에서 처리)
v10. 그리드 상에서 수정불가

v11. 환자목록을 누르면 띄웠던 패널의 내용은 사라지고 새로 폼이 생성됨.(if)
v12. 그리드에 환자목록이 나타남.
v13. 에디트에 환자명 입력 뒤 조회버튼 누르거나 엔터 누르면 검색가능.
v14. 추가버튼을 누르면 새로운 환자정보 입력가능. 입력 후 확인 누르면 추가.
v15. 취소버튼을 누르면 내용이 사라짐.
v16. 환자 선택 후 삭제 누르면 환자 정보 삭제 가능.
v17. 환자 선택하면 추가 입력창과 같은 곳에 환자 정보가 조회됨.
v18. 환자 선택 후 환자 정보 수정 후 확인 누르면 편집 가능.
19. 문자 버튼을 누르면 환자 정보와 함께 SMS 화면으로 넘어감.

20. 진료 내역 버튼을 누르면 기존의 폼 내용이 사라지고 패널에 진료 폼이 나타남.
21. 환자명, 예약일, 내원사유로 검색가능. 
22. 차트를 누르면 월별 방문수 차트로 새 창에 표현됨.
23. 환자 선택 시 진료일, 금액, 담당의사, 내원사유, 처방약품, 처방량이 포함된 진료내역이 표시됨.
24. 출력버튼 누르면 그동안 쌓인 진료내역이 출력됨(옵션)

v25. 약품목록 버튼을 누르면 띄웠던 폼내용이 사라지고 약품관리폼 생성.
v26. 약품목록을 에디트에 입력해 검색할 수 있음.
v27. 약품 추가를 누르면 새로운 약품 입력가능.
v28. 취소를 누르면 입력중인 약품내용 초기화
v29. 목록에서 약품을 선택 뒤 내용 수정 후 확인 누르면 기존 내용 갱신.
v30. 삭제를 누르면 선택한 약품 내용 삭제 가능.

31. 문자 전송 버튼을 누르면 띄웠던 폼이 내려가고 문자전송 폼이 패널에 띄워짐.
32. 환자명, 예약여부를 검색할 수 있음.
33. 앱 모양 화면(memo)에 전송할 문자내용을 입력 할 수 있음.
34. 레이디오 박스를 클릭하면 기존의 저장된 내용이 memo에 복사됨.(입력중이던 내용은 제거)
35. 환자목록에서 클릭 하거나 예약접수 폼에서 문자버튼을 눌렀을 경우에 환자가 체크됨.
36. 전송을 누르면 입력된 문자내용이 체크된 환자들에게 전송된 뒤 문자내용이 초기화 되고 전송완료 창이 나옴.
37. 초기화를 누르면 입력한 문자내용이 초기화됨.

v38. 진찰 버튼을 누르면 전체화면으로 진찰 폼이 나타남.(의사는 수납을 안하므로)
39. 진찰 중으로 상태가 변경된 환자들만 입장환자 목록에 표시.
40. 입장환자를 선택 시 환자명과 내원사유, 처방내용 등이 표시됨.
41. 이번 진찰의 내원사유를 dbmemo창에 입력 가능.
42. 약검색 Edit에 약품명을 입력 하면 실시간으로 약품명이 검색됨.
43. 약품명을 더블클릭시 처방내용에 약품명이 옮겨가고 처방량 입력가능.
44. 처방 버튼 클릭 시 처방내용과 진료일, 내원사유, 결제금액이 DB에 입력되고 현재상태가 접수중(30)으로 바뀌면서 진찰환자 내용이 초기화됨.
45. 취소 버튼을 누르면 진찰환자 입력내용이 사라짐.(입장환자 선택순간으로 돌아감)
46. 입장환자 누른 뒤 삭제 누르면 현재상태 0으로 바뀌면서 예약일과 입장 환자 삭제
47. 의사가 처방을 완료하면 수납목록에 환자표시 넘어감. (현재상태 30)
48. 수납버튼을 눌러 환자가 금액을 결제 한 뒤(결제기능 생략) 처방받은 내역들이 처방내역에 저장되고 기존 입력내용은 초기화.
49. 출력 버튼을 누르면 처방전이 출력 됨.(옵션)

 
제목 : 방문대기자 열람 프로그램
설명 : 진찰 전 대기중인 환자들 목록을 모니터링 할 수 있다.
사전 조건 : 1. 당일 예약자가 있어야함.
2. 점심시간 시 '점심시간 입니다' 문구가 출력되야 함.

테스트 절차
1. 프로그램 실행 시 메인 폼 나타남.
2. 순서대로 입장예정 환자들 목록이 표시됨.
3. 순번이 1인 환자는 커다랗게 표시됨.
4. 12시에서 2시까지 '점심시간 입니다' 문구가 출력됨.
5. 다른 곳 클릭 시 문구가 사라짐.
6. 점심 버튼 클릭 시 다시 문구가 출력 됨.
5. 접수 종료 버튼 클릭 시 '금일 접수는 종료되었습니다' 문구 출력.
6. 종료 버튼 클릭 시 프로그램 종료.

제목 : 병원 예약접수 앱
설명 : 환자는 앱으로 쉽게 병원 예약을 할 수 있고 본인의 순서를
확인하거나 알림을 TTS로 받을 수 있다.
사전조건 : 1. 환자가 앱을 이용하려면 본인의 정보를 등록해야 함.
2. 본인의 정보 외엔 열람할 수 없게 해야 함.

테스트 절차 : 
1. 앱 실행 시 회원가입 여부, 전화 권환 수락 여부 확인 후 진행
2. 회원미가입 시 성명, 생년월일, 연락처 기입하는 회원가입 창 출력.
3. 회원가입 완료 혹은 이미 가입 시 이용방법 창이 나옴.
4. 이용방법은 옆으로 탭을 넘기며 확인 가능
5. 멀티뷰 버튼 터치 시 Drawer로 나오면서 접수예약, 접수내역, 대기순서확인 순으로 리스트가 나옴.
6. 접수예약 클릭 시 예약일 과 예약시간 기입가능. 담당의사 선택가능.
7. 신청 버튼 터치 시 입력한 내용이 DB에 입력되면서 접수내역으로 이동. 현재상태 변경(10)
8. 접수내역 터치 시 본인이 예약한 내용을 확인및 취소 가능.
9. 취소버튼 터치 시 현재상태 0으로 바뀌고 예약일 예약시간 담당의사 삭제됨.
10. 병원에서 수납완료시 접수내역에서 사라짐.
11. 대기순서확인 터치 시 현재 병원에서 대기하는 예약자들 목록이 List view로 표시됨.
12. 환자 선택 후 LEFT 슬라이드 시 그 환자의 예약일과 예약시간이 표시됨.
13. RIGHT 슬라이드 시 다시 대기순서 목록으로 돌아옴.
14. 본인의 순서가 왔을 시 진동, 알림음과 함께 입장하라는 TTS 기능이 자동실행됨.
15. 스마트폰의 이전 버튼 터치 시 종료 하시겠습니까? 문구와 함께 확인, 취소 버튼이 나오고 확인시 앱 종료됨.
